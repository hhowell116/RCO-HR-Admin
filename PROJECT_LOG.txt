PROJECT OVERVIEW
================
Employee recognition display system for Rowe Casa Organics facility TVs (via
Airtame). HR (Celina Bianco) manages all content. Zero IT dependency for
day-to-day operations.

LIVE URLS

Rockstars:    https://rco-hr-display.web.app/rockstars
Celebrations: https://rco-hr-display.web.app/celebrations

Firebase Project: rco-hr-admin (Spark/free plan)
Firebase Console: https://console.firebase.google.com/project/rco-hr-admin

================================================================================
TECH STACK
================================================================================
- React 18 + TypeScript + Vite (both apps)
- Tailwind CSS (admin panel styling)
- Firebase Firestore (database — all data including photos as base64)
- Firebase Hosting (two sites: admin + display)
- Firebase Auth (Google SSO, @rowecasaorganics.com only)
- No Cloud Storage (free plan limitation — photos stored as base64 in Firestore)
- No Cloud Functions (free plan — all sync logic runs client-side)

ROLES & ACCESS (RBAC)
================================================================================
Roles stored in Firestore /users collection (not IdP claims — portable).
Only @rowecasaorganics.com Google accounts can sign in.

IT Admin:
  - Full access to everything
  - User management (assign roles)
  - IT Overview (system health, data tools, imports)
  - Delete employees & campaigns
  - First user to sign in auto-gets this role

HR:
  - Employee roster (add, edit, import CSV)
  - Birthdays & anniversaries
  - Campaigns (create, edit, activate)
  - Display settings
  - View displays
  - Default role for subsequent sign-ins

C-Suite:
  - Same as HR (executive visibility)

All Others:
  - Blocked at login (non-RCO domains rejected)

FIRESTORE SCHEMA
================================================================================
/users/{uid}
  email, displayName, role, createdAt, lastLogin, createdBy

/employees/{id}
  firstName, lastName, displayName, email, department, jobTitle,
  employeeId, phoneLast4, photoUrl (base64), photoStatus,
  birthMonth, birthDay, hireDate, location, isActive,
  createdAt, updatedAt, updatedBy

/campaigns/{id}
  type (birthday|anniversary|rockstar|custom), title, month, year,
  isActive, displayOrder, createdAt, updatedAt, updatedBy

/campaigns/{id}/entries/{id}
  employeeRef, employeeName, employeeTitle, employeeTenure,
  employeeInitials, photoUrl, quote, badgeText, isVisible,
  displayOrder, createdAt, updatedAt

/displayContent/{id}  (materialized view — synced client-side)
  type, campaignId, entryId, employeeName, employeeTitle,
  employeeTenure, employeeInitials, photoUrl, quote, badgeText,
  displayOrder, isActive, updatedAt

/displayConfig/default
  rotationSpeed, transitionDuration, showProgressBar, showDotNav, isLive

KEY FEATURES
================================================================================

AUTO-CAMPAIGN CREATION:
  When HR adds/edits an employee's birthday or hire date, the system
  automatically finds or creates the appropriate monthly campaign and
  adds the employee to it. No manual campaign creation needed.

CSV IMPORT:
  HR can download a template, fill it in, and upload. Handles:
  - Flexible column headers (BambooHR, custom, whatever)
  - Missing fields (only name required)
  - Combined "Name" column auto-split into first/last
  - Birthday parsing: separate month/day, "4/15", full dates, "DOB"
  - Duplicate detection: checks name + email against existing employees
  - Per-row error handling: one bad row won't kill the import
  - Auto-creates campaigns for any birthdays/hire dates in the data

DISPLAY APP:
  - /              All content combined (rockstars + birthdays + anniversaries)
  - /rockstars     Rockstar slides only (rotating)
  - /celebrations  Calendar board with birthday/anniversary cards (static view)
  - Real-time updates from Firestore
  - Fullscreen kiosk-ready (press F or click)
  - Keyboard nav (arrows, spacebar)
  - Configurable rotation speed, progress bar, dot nav

GUIDED TOUR:
  - Auto-launches on first visit
  - "Retake Tour" button in sidebar
  - Anchors to sidebar nav items with arrow + gold highlight
  - Walks through all features with descriptions
  - Remembered per-browser via localStorage

AUTH ARCHITECTURE (IdP-Agnostic)
================================================================================
Auth provider determined by VITE_AUTH_PROVIDER env var:
  "google" (current) — Google Workspace SSO
  "microsoft" (future) — Microsoft 365 / Entra ID

Switching IdPs requires:
  1. Change env var
  2. Enable new provider in Firebase console
  3. Map old UIDs to new UIDs by email in /users collection
  Roles survive because they're in Firestore, not IdP claims.

DECISIONS & TRADE-OFFS
================================================================================

1. PHOTOS AS BASE64 IN FIRESTORE
   Why: Firebase free plan doesn't include Cloud Storage.
   Trade-off: Larger Firestore documents (~15-30KB per photo at 200-300px).
   Works fine for <500 employees. If roster grows significantly, upgrade to
   Blaze plan and move to Cloud Storage.

2. CLIENT-SIDE DISPLAY SYNC (NO CLOUD FUNCTIONS)
   Why: Cloud Functions require Blaze plan.
   Trade-off: The admin panel writes to /displayContent directly when
   campaigns are created/updated. If two admins edit simultaneously,
   last write wins. Acceptable for HR team of 1-3 people.

3. SEPARATE FIREBASE PROJECT
   Original plan was to reuse rco-rockstars project. Changed to rco-hr-admin
   to keep the existing Rockstars site untouched.

4. CELEBRATIONS BOARD vs ROTATING SLIDES
   Birthday/anniversary display uses a calendar board layout (left: calendar,
   right: cards) instead of rotating individual slides. Better for persistent
   wall displays where people walk by and read.
